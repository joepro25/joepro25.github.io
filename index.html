ResDesk - Babcock Canada

This is the home page that a user would first land on.

The user can search for available desks, based on the criteria they provide. The desks will be displayed in a list of cards.

The user can look at a map, to see where the desks are located.

When the user confirms their booking, it will be added to the list of bookings on their profile.

The Manager Menu allows a team manager to select multiple employees, and create a group booking for them.

A list of desks in the booking will be returned. They are blocked out for 5 minutes, to allow the manager to confirm the booking. If the time runs out, the desks are released again to be reserved by others.

Blackout periods can be created to block out one or more desks for a period of time, which they are unavailable. For example, this could be used for if there are any renovations happening around the desk. Blackouts can be edited to change the dates, or to add or remove affected desks.

The Administration Menu is for office admins to edit office locations, desk type, zone types, and add, edit, or remove desks from the application.

After the core functionality was finished, I started working on an interactive map, which would allow users to select their desk from a map. The user could create a search, and the map would only display desks, meeting the criteria.

When a desk is selected, the square turns green. I did not have time to fully implement booking with the interactive map.

In the Create Desks page for the map, admins could enter the details of a desk into the form, then select a location on the map. Clicking on the map would add the coordinated from the HTML canvas to the x and y fields in the form. The admin can then submit the form.

Here, the new desk has been added on the map at desk "OTT1008".

Othello - Java
Overview

Othello (type of Reversi) is a board game, played on the same board type as chess/checkers. The game starts with two pieces for each player, in the centre four squares, diagonal from each other. The players take turns "capturing" the other player's pieces, by surrounding one or more of the other player's pieces. The captured pieces switch to the current player's colours. When there are no available moves left, the game is over. Whichever player has the move pieces, wins. Full rules can be found here .

This game was created for my Level 4 class at Algonquin, Java App Programming. It was written in Java, using JavaFX to build the GUI. The game also has a simple socket server, for chatting between two or more clients. The server however does not support online play, only local (hot seat) play is supported.

Othello Features
Main Game
  • Supports creating a new game at any time, resets the board, and resets all scores
  • Gives feedback after every turn, to let players know whose turn it is, and how many pieces were captured
  • Displays the score in the bottom right, and highlights the player whose turn it is
  • Valid moves button, which will display check marks where there are valid moves for the player
  • Game pieces can be changed. Styles include:
    • Regular pieces (black and white chips)
    • Dogs vs. Cats
    • Bats vs Pumpkins
Chat Client
  • Allows multiple clients to join into a chat room
  • Send messages to and see messages from all clients in the room
  • Enforces unique names on the server
  • Slash commands:
    • /help: Show all of the slash commands
    • /bye: Disconnect from server
    • /name (name): Rename yourself
    • /emote (text): Send an emote
    • /pvt (user name) (message): Send a private message to the specified user
Third Project
Overview

This application is a banking simulator, written in Java 8, using JavaFX FXML for the GUI. This simulator provides functionality for creating accounts (chequing/saving), updating accounts (deposit/withdrawal), displaying accounts by account number, printing all accounts to console, and outputting to file, reading account records from file, and running the monthly update for all the accounts (charge fees, add interest).

Features
Add Account
  • Allows creating either a chequing or savings account
  • Both account types require basic personal info
  • Chequing requires a monthly fee
  • Savings requires a minimum balance and an interest rate
Update Account
  • Allows either a deposit or withdrawal to be made to or from an account
  • A withdrawal will be disallowed if there are not enough funds in the account
Display Account
  • Allows user to search for an account by its ID, and will display the account's information
Print Accounts
  • Prints the list of accounts and details to console, and outputs to a file
Find Account Index
  • Returns the array index of the searched account number
Read Records
  • Reads account records from a text fields
Run Monthly Update
  • Charges the monthly fees on chequing accounts
  • Deposits the monthly interest on savings accounts
  • Does not charge the monthly fee if there are not enough funds in the chequing account
  • Does not deposit interest if the minimum balance is not met on a savings account
