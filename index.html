ResDesk - Babcock Canada

This is the home page that a user would first land on.

The user can search for available desks, based on the criteria they provide. The desks will be displayed in a list of cards.

The user can look at a map, to see where the desks are located.

When the user confirms their booking, it will be added to the list of bookings on their profile.

The Manager Menu allows a team manager to select multiple employees, and create a group booking for them.

A list of desks in the booking will be returned. They are blocked out for 5 minutes, to allow the manager to confirm the booking. If the time runs out, the desks are released again to be reserved by others.

Blackout periods can be created to block out one or more desks for a period of time, which they are unavailable. For example, this could be used for if there are any renovations happening around the desk. Blackouts can be edited to change the dates, or to add or remove affected desks.

The Administration Menu is for office admins to edit office locations, desk type, zone types, and add, edit, or remove desks from the application.

After the core functionality was finished, I started working on an interactive map, which would allow users to select their desk from a map. The user could create a search, and the map would only display desks, meeting the criteria.

When a desk is selected, the square turns green. I did not have time to fully implement booking with the interactive map.

In the Create Desks page for the map, admins could enter the details of a desk into the form, then select a location on the map. Clicking on the map would add the coordinated from the HTML canvas to the x and y fields in the form. The admin can then submit the form.

Here, the new desk has been added on the map at desk "OTT1008".

ResDesk Features
Main Screen:
  • Search for available desks based on criteria such as type and date range
  • View map of desks in a modal to check where desk is located
  • Confirm a desk booking
  • Review and edit or delete bookings
  • Add unavailable desks to a waitlist, to be notified if it becomes available
  • Change default user preferences for searching desks
Manager View:
  • Select multiple employees, and create group booking
  • Hold desks for 5 minutes for chance to review and confirm booking
  • Relese desks in time runs out
Blackout View:
  • Create reseration blackouts on one or more desks (e.g. renovations)
  • Desks will be unavailable for booking in the time period
  • Edit blackouts, to changes the dates of the blackouts, or to add or remove desks from a blackout
Maintenance View:
  • Edit locations, desk types, zone types (e.g. security clearances)
  • Add, edit and delete desks
  • Hold desks for 5 minutes for chance to review and confirm booking
  • Relese desks in time runs out
Interactive Map:
  • Search for available desks based on criteria entered
  • Display available desks meeting the criteria on the map
  • User can select a desk on the map, and the box will turn green
  • I did not have time to fully implement the booking feature on the map
Interactive Map Maintenance:
  • Create desks with specified features
  • Click on the map to select the x and y cooridnates of the map
  • Submit the new desk, and have it appear on the map
